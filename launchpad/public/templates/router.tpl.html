Single-Page Application Router

Simple Demo Application

A simple web-application with a few static pages. This is the first demo site I created, with little or no concern for advanced functionality. The pages contain primarily text-based templates. None of the routes are dynamic in any way. The "Page Not Found" route was retrofitted after it was intially added to the capstone router.

Programmatic Navigation (With 404)

A test bed created to showcase programmatic page navigation, including routing to a catchall page when a page is not found. Catchall routes can be used for more than just "404 not found" errors. Use the form, table of contents, and history on the left to test each of the routes.

Heavy Bundle Simulation

A simple static site including both lightweight and heavy pages. Included to showcase the both the lazy-loading and caching of expensive navigation operations. When the user navigates to the pi page, which contains 1,000,000 digits of pi, several megabytes of text are downloaded before the route navigation completes. The next time the user visits the page, the text itself does not need to be downloaded again, although there is still a slight delay for rendering.

Route Parameter Demo

A demo using route parameters and and template factories to dynamically construct the template for the page the user has requested. There are actually only three routes defined, although that allows for any number of pages through the use of route parameters. Interfaces with the GitHub API to demonstrate promise-based template factories.

Synchronous Development Spike
